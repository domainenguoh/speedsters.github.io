Cool Research Project

Analyzing the Correlation between Database Schema Complexity and GitHub Activity

Group Members: Rani Maklada, Neil Hamza, Jeries Shomaly, Bashar Yousef

About the Project

Welcome to our cool research project! In this study, we aim to investigate the relationship between the complexity of a project's database schema and its GitHub activity. We hypothesize that as the complexity of the database schema increases, there will be a corresponding rise in the number of commits and issues on GitHub. This correlation can provide valuable insights into the challenges and maintenance efforts involved in managing such projects.

Our research focuses on analyzing a sample of Java and Python projects on GitHub, extracting data using Gitana, and visualizing the database schemas with DBDiagram.io. By comparing the complexity of the schemas with GitHub activities, we seek to shed light on how schema design impacts project development and management.

Tools Used

For our research, we employed two powerful tools:

  • Gitana: A data extraction tool that helps us collect code, commit histories, and issue data from Java and Python projects on GitHub.
  • DBDiagram.io: A visual database design tool that allows us to generate diagrams representing the complexity of the database schemas of each project.
