Data Schema Complexity Analysis

Exploring the Correlation between Data Schema Complexity and GitHub Activity

Group Members: Rani Maklada, Neil Hamza, Jeries Shomaly, Bashar Yousef

About the Project

Welcome to our fascinating research project! Here, we delve into the intriguing relationship between the complexity of a project's database schema and its GitHub activity. Our hypothesis is that as the complexity of the database schema increases, there will be a corresponding rise in the number of commits and issues on GitHub. This correlation can provide valuable insights into the challenges and maintenance efforts involved in managing such projects.

Our research focuses on analyzing a diverse set of projects hosted on GitHub. By searching using GitHub API, extracting data using Gitana, visualizing database schemas with DBDiagram.io, and analyzing images with OpenCV, we strive to gain deeper insights into how schema design impacts project development and management.

Research Workflow

Our research follows a systematic workflow that involves multiple stages:

  1. Use GitHub API to search for repositories with .dbml files.
  2. Store the repository details in a TXT list for further processing.
  3. Clone each repository locally using Git to analyze it further.
  4. Utilize Gitana to extract commit histories, issues, and user data from Java and Python projects.
  5. Store the extracted data in a MySQL database for each repository.
  6. Locate the DBML files in each repository and upload them to DBDiagram.io using Selenium.
  7. Extract the data structure image and analyze its complexity using OpenCV.
  8. Analyze the repository complexity based on metrics like commits and issues using MySQL data.
  9. Establish a correlation between data schema complexity and repository complexity.

Through this workflow, we aim to unveil the impact of data schema complexity on project development and management on GitHub.

Tools Used

For our research, we utilized the following powerful tools:

  • GitHub API: We use the GitHub API to search for repositories containing .dbml files on GitHub. The API allows us to extract the list of files, along with the owner and repository names.
  • Gitana: A data extraction tool to collect code, commit histories, and issue data from Java and Python projects on GitHub.
  • DBDiagram.io: A visual database design tool for generating diagrams representing the complexity of database schemas.
  • Selenium: To automate the process of uploading DBML files to DBDiagram.io.
  • OpenCV: For image analysis to measure complexity based on shape, size, and line count.
  • MySQL: To store and analyze the extracted data for each repository.
